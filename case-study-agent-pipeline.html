The Problem

Deployment OS proved the spine: catch a signal, score it, route it. But the head and tail were still partly manual — something had to feed the signals in, and something had to act on the output. The next problem was closing that loop: building the layer that runs the full outbound motion end to end, with no human in the pipe until a draft is ready for review.

The risk with autonomous systems is obvious — an agent that can act without oversight can also do damage at scale. So the real challenge wasn't building agents that work. It was building agents that are safe: restartable, auditable, and structurally incapable of doing the wrong thing.

The Build

A pipeline of focused agents, each doing exactly one job, coordinating only through a shared database. No agent talks to another agent directly — they read rows in one status, do their work, and write rows in the next status. The database is the assembly line; the agents are the stations.

  • Scout — detects hiring signals from public ATS job-board APIs. Config-driven (swap a config to target a different ICP), multi-source (Greenhouse, Lever) with normalized adapters, deduplication, and per-source failure tracking so misses are visible, not silent.
  • Enricher — resolves the right contact at each company, running a provider waterfall (multiple data sources tried in priority order) with confidence scoring, so contacts are verified, not guessed. Hands off to the existing scoring spine.
  • Synthesizer — drafts personalized outreach with an LLM, anchored on the specific signal. Sandboxed: it can only write drafts to a pending queue. It has no send capability whatsoever — every draft is held for quality review.
  • QA + Dispatch (in progress) — a deterministic quality gate that approves or kills each draft, and a rate-limited dispatcher that only ever sends approved, gated copy.
The Principle

Agents handle the judgment work; deterministic code handles everything else. The system knows where not to put an agent. Every handoff is a database row you can inspect. Any agent can die and restart with no lost state.

The one agent that uses an LLM is the one most able to go wrong — so it's the one most tightly caged. It can write. It cannot send.

The Metrics

What this is built to move:

  • Full-motion automation — signal to ready-to-review draft with zero manual steps.
  • Data quality — percentage of contacts verified across two sources vs. single-source guesses.
  • Safety — zero un-reviewed copy can reach a send channel, by construction.
  • Throughput — signals processed per run, with full per-step auditability.
